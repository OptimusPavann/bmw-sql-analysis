# BMW Sales SQL Analysis

## Project Overview
This project analyzes a BMW vehicle sales dataset using SQL.  
The goal is to practice and demonstrate SQL skills by solving 50 business-oriented analytical questions.

The project covers multiple SQL concepts ranging from basic queries to advanced analytical techniques such as subqueries, window functions, and common table expressions (CTEs).



## Dataset Description

The dataset contains information about BMW vehicle sales across different regions and years.

**Columns included:**

- model
- year
- region
- color
- fuel_type
- transmission
- engine_size_l
- mileage_km
- price_usd
- sales_volume

Each row represents a summarized sales record for a specific vehicle configuration.


## Project Structure

The SQL queries are organized into different sections based on difficulty and concepts.

### Part 1 — Basic SQL Queries
Basic exploration queries including:
- counting records
- identifying unique values
- basic aggregations

### Part 2 — Aggregation Analysis
Queries using:
- `SUM`
- `AVG`
- `COUNT`
- `GROUP BY`

### Part 3 — GROUP BY + HAVING
Filtering aggregated results using:
- `HAVING`
- aggregated conditions

### Part 4 — Business Analysis Queries
Business-oriented analytical problems such as:
- top selling models
- least selling vehicles
- regional performance

### Part 5 — Subquery Problems
Advanced filtering using:
- scalar subqueries
- comparisons with overall averages

### Part 6 — Window Functions
Analytical SQL techniques using:
- `RANK`
- `DENSE_RANK`
- `ROW_NUMBER`
- running totals

### Part 7 — Common Table Expressions (CTE)
Complex analytical queries using:
- CTEs
- window functions with ranking
- multi-step transformations
